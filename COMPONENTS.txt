6. SCROLL REVEAL SYSTEM
═══════════════════════════════════════════════════════════════════

Two observer pattern — regular elements vs. tiles/stats.

BASE CLASS
  .r        Pre-reveal state: opacity:0, translateY(30px).
            JS adds .on when element enters viewport.
            .r.on → opacity:1, transform:none, transition fires.

LATE TRIGGER CLASS (tiles and stat cards only)
  .r-late   Use alongside .r. Watched by obsTile (tighter rootMargin)
            instead of the standard obs observer.
            Tiles only animate when the tile itself is in view,
            not when the section header enters viewport.

DELAY CLASSES (stagger)
  .d1       0.12s     .d4   0.84s
  .d2       0.40s     .d5   1.04s
  .d3       0.62s     .d6   1.24s

REVEAL VARIANT RULES
  Eyebrow (.ch, .ch-d)    clip-path: inset(0 0 0 100%) → inset(0)
                          direction: left-to-right reveal
  Title (.st, .st-d)      duration: 1.9s, ease-reveal easing
  Body (.bp, .bp-d)       blur-to-clear filter, 1.7s
  Tile / stat card        @keyframes tile-reveal (see below)

TILE REVEAL ANIMATION
  Applied to: .hiw-tile.r.on, .phi-stat-card.r.on
  Uses @keyframes tile-reveal — NOT transition — so it runs
  independently of the hover spring transition.

  @keyframes tile-reveal {
    from { opacity: 0; transform: translateY(36px); }
    to   { opacity: 1; transform: none; }
  }

  animation-fill-mode: backwards — keeps tile invisible during
  its delay period. No forwards fill — .r.on CSS owns final state
  so hover lift works normally after reveal completes.

  Tile stagger delay overrides (high-specificity rules):
    .hiw-tile.r.on.d3, .phi-stat-card.r.on.d3  → 0s (immediate)
    .hiw-tile.r.on.d4, .phi-stat-card.r.on.d4  → 0.15s
    .hiw-tile.r.on.d5, .phi-stat-card.r.on.d5  → 0.30s
    .hiw-tile.r.on.d6, .phi-stat-card.r.on.d6  → 0.45s

JS OBSERVERS
  obs       rootMargin: -30px. Watches all .r:not(.r-late).
  obsTile   rootMargin: -12%. Watches all .r-late.
            Fires only when the tile is in the lower viewport.

USAGE EXAMPLES
  Section heading:   <h2 class="st r d1">
  Body paragraph:    <p class="bp r d2">
  HOW I WORK tile:   <div class="hiw-tile r r-late d4">
  Philosophy stat:   <div class="phi-stat-card r r-late d5">

9. CANVAS DOT RIPPLE (interactive)
═══════════════════════════════════════════════════════════════════

Cursor pushes nearby dots away; they lerp back when cursor moves.
Applied to: .contact-zone, .philosophy-sec#work (MADE REAL).

HTML required inside the section:
  <canvas class="cz-canvas" aria-hidden="true"></canvas>
  <div class="cz-dots-static" aria-hidden="true"></div>

JS: initDotRipple(element) — defined once, called per section:
  initDotRipple(document.querySelector('.contact-zone'));
  initDotRipple(document.getElementById('work'));

Parameters (currently hardcoded in JS):
  SPACING = 32px grid
  DOT_R   = 1.5px radius
  DOT_COLOR = rgba(255,255,255,0.10)
  RIPPLE_RADIUS = 150px
  MAX_DISPLACE  = 18px
  LERP          = 0.12 (snap-back speed)

Mobile (pointer:coarse): canvas hidden, .cz-dots-static shown.
Desktop: section backgroundImage set to 'none' (canvas replaces it).

CSS:
  .cz-canvas      position:absolute; inset:0; z-index:0.
  .cz-dots-static position:absolute; inset:0; display:none by default.

═══════════════════════════════════════════════════════════════════
15. INTERACTION PATTERNS
═══════════════════════════════════════════════════════════════════

CANVAS DOT RIPPLE
  Cursor displaces nearby dots (within 150px radius).
  Math: displacement = (radius - dist) / radius * maxDisplace
  Dots lerp back: offset += (target - offset) * 0.12
  Implemented in initDotRipple(el). Pass any section element.

TWO-FACE CARD (how-i-work tiles)
  Default face fades out + translates up (-3px on hover).
  Hover face fades in + translates up from 10px offset, 0.1s delay.
  Fixed card height (260px) — no layout shift ever.
  Title is duplicated in both faces (one per face in HTML).
  Hover content: padding-left/right 2.2rem, body text 12pt.

SVG BORDER TRACE
  SVG <rect> with pathLength="1000" normalization.
  stroke-dashoffset: 1000 → 0 on hover.
  Smooth ease-in-out cubic-bezier, no bounce.

COPY TO CLIPBOARD
  .cz-copy-btn click → navigator.clipboard.writeText()
  Adds .copied class for 2000ms → icon swaps copy → check.
  Fallback: textarea + execCommand for older browsers.

SCROLL-SPY NAV
  IntersectionObserver watches all section[id] elements.
  Active link class applied to nav <a> matching section id.
  Threshold: whichever section is most visible gets highlighted.

SCROLL REVEAL (two-observer system)
  obs: rootMargin -30px, threshold 0.1 — fires for .r:not(.r-late)
  obsTile: rootMargin -12%, threshold 0.15 — fires for .r-late
  Both add .on class; tile reveal uses @keyframes, others use transition.

CUSTOM CURSOR
  5 states tracked via JS mousemove + hover detection.
  Lerp ring follows cursor with lag for smooth trailing effect.

HERO ORBIT JS
  Single orbit ring. JS handles rotation only.
  animOrbits(): oA += SPEED1 * speedMod
  orbit.style.transform = `translate(-50%,-50%) rotate(${deg1}deg)`
  Mouse proximity slows rotation (influence based on distance).
  Scale is NOT in JS transform — handled exclusively by .orbit-shell CSS.
