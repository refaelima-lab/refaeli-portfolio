CURSOR RIPPLE EFFECT ON DOT GRID
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━


On mousemove over the contact section background:
Draw the dot grid on a Canvas element instead of CSS background-image.
Each dot's position is displaced by a ripple/wave distortion 
based on distance from the cursor.


Distortion math:
- Each dot calculates its distance from cursor (mx, my)
- If distance < rippleRadius (120px): 
  dot is pushed away from cursor proportionally
  displacement = (rippleRadius - dist) / rippleRadius * maxDisplace
  maxDisplace = 10px
- The dot snaps back smoothly when cursor moves away
  Use lerp on each dot's offset: offset += (target - offset) * 0.12


This creates a ripple/distortion that follows the cursor —
like pressing on a flexible surface. Dots near cursor push outward,
dots far away are undisturbed.


The effect should be subtle — the grid reads as normal until 
you move the cursor over it, then it feels alive.


On mobile / touch: disable the canvas distortion, 
fall back to static CSS dot grid.
